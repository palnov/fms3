# Дизайн-документ: ИИ-консультант по миграционному праву с RAG-поиском и лидогенерацией

Этот документ описывает архитектуру, структуру данных, механизмы защиты и лидогенерации для интеграции интеллектуального консультанта на портал "Миграция в Россию".

---

## 1. Архитектура и Стек технологий

Для обеспечения максимальной скорости, автономности и отсутствия затрат на внешнюю инфраструктуру БД RAG-система будет построена на базе локального хранилища:

- **База данных**: Файл `knowledge.db` (SQLite). Используется для хранения текстовых фрагментов (чанков), их векторных представлений и метаданных шаблонов.
- **Парсинг документов**: `pdf-parse` (для `.pdf`), `mammoth` (для `.docx`), `@types/node` для файловых операций.
- **Векторизация и ИИ**: Официальное API Google Gemini (модели `text-multilingual-embedding-002` для эмбеддингов и `gemini-2.5-flash` для генерации ответов).
- **Интерфейс**: Встроенный чат-компонент React на базе Tailwind CSS, интегрируемый как на отдельную страницу (`/tools/ai-consultant`), так и в плавающий виджет.

---

## 2. Структура Базы Данных (SQLite)

В базе `knowledge.db` создаются две основные таблицы:

### Таблица `chunks` (Текстовые фрагменты законов и статей)
- `id` (INTEGER PRIMARY KEY AUTOINCREMENT)
- `content` (TEXT): Текстовый фрагмент (~1000-1500 символов).
- `source_file` (TEXT): Путь к исходному файлу (например, `docs/knowledge/fz-115.txt`).
- `embedding` (BLOB / TEXT): Векторное представление текста (768-мерный массив чисел в формате JSON-строки или бинарного блоба).

### Таблица `templates` (Шаблоны и образцы документов)
- `id` (INTEGER PRIMARY KEY AUTOINCREMENT)
- `title` (TEXT): Название документа (например, "Заявление на РВП по браку").
- `file_path` (TEXT): Путь к файлу для скачивания (например, `/templates/rvp-brak-blank.docx`).
- `sample_path` (TEXT): Путь к образцу заполнения (например, `/templates/rvp-brak-sample.pdf`).
- `keywords` (TEXT): Ключевые слова для быстрого текстового поиска (FTS) через запятую.

---

## 3. Скрипт индексации (`scripts/index-knowledge.ts`)

Скрипт запускается разработчиком вручную при изменении базы знаний или автоматически на этапе сборки проекта:

1. Читает все файлы из папки `knowledge/articles/`.
2. Извлекает текст в зависимости от расширения файла (txt, pdf, docx).
3. Разбивает текст на чанки по 1000 символов с перекрытием 200 символов.
4. Отправляет каждый чанк в Google Gemini Embedding API, получает 768-мерный вектор.
5. Сохраняет чанк, источник и вектор в таблицу `chunks`.
6. Читает файл-конфиг шаблонов `knowledge/templates-map.json` и заполняет таблицу `templates`.

## 4. Алгоритм работы API-роута (`/api/consultant`)

При получении POST-запроса от клиента с вопросом пользователя:

1. **Проверка Лимитов**: Сервер проверяет IP-адрес пользователя и его куки. Если лимит (20 запросов в сутки) исчерпан, возвращается статус 429 с предложением связаться с юристом напрямую.
2. **Защита (Prompt Injection Guard)**:
   - Быстрая проверка вопроса на сервере регулярными выражениями на наличие подозрительных команд ("ignore previous instructions", "write code", "system prompt").
   - Векторизация вопроса пользователя через Google Embeddings API.
3. **Поиск контента (Retrieval)**:
   - Вектор вопроса сравнивается со всеми векторами в `knowledge.db` (расчет косинусного сходства). Извлекается топ-4 самых похожих чанка.
   - Делается текстовый поиск по таблице `templates` по совпадению ключевых слов с вопросом пользователя.
4. **Формирование промпта (System Prompt)**:
   - Модели передается жесткая инструкция:
     ```text
     Ты — дежурный миграционный юрист портала "Миграция в Россию".
     Отвечай ТОЛЬКО на вопросы, связанные с получением РВП, ВНЖ, гражданства РФ, патентов, виз и миграционного учета.
     Если вопрос не относится к этой теме, вежливо откажись отвечать.
     Используй для ответа только предоставленный контекст.
     Если в контексте есть ссылки на шаблоны документов, обязательно предложи их пользователю в markdown формате: [Название](ссылка).
     Всегда в конце ответа ненавязчиво, но убедительно предлагай связаться с живым юристом для бесплатной экспресс-проверки ситуации, используя специальный тег [CTA_LAWYER_FORM].
     ```
5. **Генерация ответа**: Модель `gemini-2.5-flash` формирует ответ. Если в конце ответа присутствует тег `[CTA_LAWYER_FORM]`, клиентское приложение отрендерит интерактивную форму сбора лидов прямо внутри чата.

---

## 5. Интеграция Лидогенерации (CPA Правовед)

Когда пользователь заполняет форму сбора лидов в чате:
1. Данные (Имя, Телефон, Вопрос) отправляются на внутренний эндпоинт `/api/leads`.
2. Сервер валидирует телефон (11 цифр, начинается с 7).
3. Отправляет лид по API в партнерскую сеть Feedot/Pravoved.
4. В чате выводится сообщение: *"Ваша заявка успешно отправлена дежурному юристу. Ожидайте звонка в течение 15 минут!"*.
