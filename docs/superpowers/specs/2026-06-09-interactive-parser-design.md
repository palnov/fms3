# Дизайн-документ: Локальный интерактивный парсер ресурсов для RAG-системы

Этот документ описывает архитектуру, структуру данных и логику работы локального парсера, предназначенного для скачивания веб-страниц и документов, их сохранения на локальном сервере и индексации в RAG-системе с сохранением ссылок на страницы-источники.

---

## 1. Цель и контекст использования

Поскольку многие официальные российские миграционные и правовые ресурсы (МВД, Госуслуги, правовые порталы) блокируют запросы из-за рубежа (включая IP-адреса хостингов, где работают LLM и облачные парсеры), парсинг контента должен производиться **локально** на машине разработчика/администратора. 

При этом RAG-система должна:
1. Хранить контент и сами файлы локально на сервере, чтобы избежать битых ссылок в будущем.
2. Ссылаться на оригинальную страницу-источник (где был размещен документ/статья), чтобы пользователь мог найти обновленную информацию, если файл устареет.

---

## 2. Структура Базы Данных (SQLite)

В существующей базе `knowledge.db` таблица `chunks` расширяется новыми колонками для хранения метаданных источника.

### Схема таблицы `chunks`
```sql
CREATE TABLE IF NOT EXISTS chunks (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  content TEXT NOT NULL,
  source_file TEXT NOT NULL,       -- Имя локального файла (например, "instruction_rvp.pdf")
  source_url TEXT,                 -- Ссылка на родительскую страницу-источник (например, "https://mvd.ru/laws/page")
  local_download_url TEXT,         -- Ссылка для скачивания файла с нашего сайта (например, "/downloads/instruction_rvp.pdf")
  embedding TEXT NOT NULL          -- Вектор эмбеддинга в формате JSON-строки
);
```

## 3. Архитектура Интерактивного Парсера

Парсер реализуется как интерактивный CLI-скрипт `scripts/interactive-parser.ts`.

### Логика работы:
1. **Запуск**: Разработчик запускает скрипт, передавая адрес страницы:
   ```bash
   npx tsx scripts/interactive-parser.ts "https://mvd.gov.ru/mvd/structure1/Glavnie_upravlenija/guvm/grazhdanstvo"
   ```
2. **Анализ страницы**: 
   * Скрипт скачивает HTML-код страницы.
   * Использует библиотеку `cheerio` для поиска ссылок (`<a>`) на документы (`.pdf`, `.docx`, `.doc`, `.txt`) и другие целевые статьи (`.html`).
3. **Интерактивный выбор**: 
   * Выводит пользователю список найденных документов с их названиями и URL.
   * Запрашивает через консольный ввод (readline), какие документы скачать (например: `1,3,5` или `all`).
4. **Скачивание и кэширование**:
   * Скачивает выбранные файлы в директорию `public/downloads/` с безопасным именем (транслитерация/хэш).
   * Если это HTML-страница, очищает ее от мусора (удаляет теги `<nav>`, `<header>`, `<footer>`, `<script>`, `<style>`) с помощью `cheerio`, извлекает основной текст и сохраняет его как `.txt` файл.
5. **Обновление манифеста (`knowledge/downloads-manifest.json`)**:
   * Для каждого скачанного файла добавляет или обновляет запись с метаданными:
     ```json
     {
       "mvd_grazhdanstvo_blank.docx": {
         "original_url": "https://mvd.gov.ru/files/blank.docx",
         "parent_page_url": "https://mvd.gov.ru/.../grazhdanstvo",
         "download_path": "/downloads/mvd_grazhdanstvo_blank.docx",
         "title": "Бланк заявления на гражданство"
       }
     }
     ```

---

## 4. Обновление индексатора (`scripts/index-knowledge.ts`)

Индексатор адаптируется для поддержки новых колонок в БД и чтения манифеста скачанных файлов:
1. Читает `knowledge/downloads-manifest.json` (если существует).
2. При индексации файлов из `knowledge/articles/` и `public/downloads/`:
   * Ищет имя обрабатываемого файла в манифесте скачанных файлов.
   * Если файл найден: записывает в БД `source_url` (как `parent_page_url`) и `local_download_url` (как `download_path`).
   * Если файл не найден (это локальный файл без веб-источника): записывает `NULL` в поля `source_url` и `local_download_url`.

## 5. Изменения в API и Интерфейсе RAG

### API `/api/consultant/route.ts`
При поиске релевантных чанков API формирует массив `sources`. Вместо строк с именами файлов, он возвращает структурированный массив объектов:
```json
"sources": [
  {
    "name": "Инструкция по заполнению РВП",
    "parent_url": "https://mvd.ru/documents/rvp-info",
    "download_url": "/downloads/instruction_rvp.pdf"
  }
]
```

### Фронтенд `/tools/ai-consultant/page.tsx`
Интерфейс чата отображает источники внизу ответа ИИ:
* Выводится имя источника.
* Если есть `download_url`, рядом выводится иконка скачивания со ссылкой на локальный файл (раздается из `public/downloads/`).
* Если есть `parent_url`, выводится иконка внешней ссылки, ведущая на родительский сайт-первоисточник.
